How it works

Plan first, then execute

Cogent starts in Plan mode — it reads your code and thinks before touching anything. When the plan looks right, switch to Confirm and let it work.

Shift+Tab to cycle: Plan → Confirm → YOLO → Terminal → Plan Works while idle or while the agent is running.

Four permission modes

Plan explores read-only with extended thinking. Confirm asks before writes. YOLO auto-approves. Terminal pauses the agent for manual shell access.

Multi-provider

Anthropic, OpenAI, Gemini, OpenRouter. Switch with Ctrl+M mid-session. Each tab can use a different model.

Ctrl+T Ctrl+W Shift+←→ Alt+1…9

Tabbed sessions

Multiple conversations at once. Sessions auto-persist and restore on launch. Rename with /rename.

Sub-agent delegation

The dispatch tool runs subtasks in parallel, each with its own context window. Confirmations route back to your tab.

#!/bin/bash # @tool linear_issues # @description List Linear issues assigned to me # @param status string "filter by status" # @env LINEAR_API_KEY required "Linear API key" # @env LINEAR_USERNAME required "Linear username" # @noconfirm INPUT=$(cat) STATUS=$(echo "$INPUT" | jq -r '.status // empty') curl -s -X POST https://linear.app/api/graphql \ -H "Authorization: $LINEAR_API_KEY" \ -H "Content-Type: application/json" \ -d "{\"query\": \"{ issues(filter: {assignee: …}) { nodes { title } } }\"}"

Custom tools — no MCP, just scripts

Executable scripts with @ directives in comments become agent tools. Any language — bash, Python, Node, whatever has a shebang. Project-local tools go in .cogent/tools/, global ones in ~/.cogent/tools/. Env vars live in .cogent/.env.

No protocol to implement, no server to run, no JSON schema to write. Input arrives as JSON on stdin, output goes to stdout. Mark read-only tools with @noconfirm — destructive ones require approval by default.

The agent can build its own tools. Ask it to write a custom tool and it will create the script, add the directives, make it executable, and use it in the same session. Your team's workflows become agent capabilities without any plumbing.

AGENTS.md

Put project-specific instructions in an AGENTS.md file. Cogent loads all of them from cwd to root, so monorepo-level context and package-level specifics both apply.

Three UI modes

Full-screen TUI for interactive work. REPL for simplicity. Headless for CI, scripts, and pipes — auto-approves everything.

Get started

Three commands

Go 1.24+ and at least one provider API key.

# Set a provider key
export ANTHROPIC_API_KEY="sk-ant-..."

# Install
git clone https://github.com/cogent-agent/cogent
cd cogent && make install

# Run
cogent

Keys can also go in ~/.cogent/settings or .cogent/settings per project. Also supports OPENAI_API_KEY, GEMINI_API_KEY, OPENROUTER_API_KEY.
